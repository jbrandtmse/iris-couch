## What Didn't Go Well / Challenges

### 1. Quit vs Return in Nested Try/Catch — Third Occurrence
**What happened:** Story 4.1's POST body parse error handler used `Quit` in an inner catch block. This only exited the inner try/catch, not the outer one, causing `tBodyObj` to be undefined when accessed later — resulting in 500 instead of 400.
**Root cause:** ObjectScript's `Quit` behavior in nested blocks remains a recurring trap. Despite the Quit-in-loop guideline from Epic 3, the nested try/catch variant wasn't explicitly covered.
**Impact:** Code review caught and auto-resolved it.
**Mitigation:** Expand the guideline: **`Return $$$OK` (not `Quit`) to exit the method from any nested block (inner try/catch, inner loops, inner if blocks)**. `Quit` should only be used at the top level of the outer Try block.

### 2. $System.Event API Discovery Cost
**What happened:** Story 4.2 required significant debugging to discover that `$System.Event` syntax doesn't work in compiled class methods, that event names must use global-reference format, and that `Signal()` throws on undefined resources.
**Root cause:** IRIS documentation doesn't clearly distinguish terminal-context vs compiled-class syntax for system utilities. The `$System.Event` shorthand works in terminal but fails silently or with cryptic errors in `.cls` files.
**Impact:** Extended development time for Story 4.2 (most complex story in the epic).
**Mitigation:** Document in project rules: `##class(%SYSTEM.Event)` for compiled classes, global-reference-format event names, and `Defined()` guard pattern.

### 3. Event Name Pattern Duplicated in 4 Locations
**What happened:** The event resource name string `"^IRISCouch.LPChanges(""" _ pDB _ """)"` is constructed identically in DocumentEngine.Save, SaveDeleted, SaveWithHistory, and ChangesHandler.HandleChanges.
**Root cause:** Event signaling was added to 3 existing methods (DocumentEngine) and consumed by 1 new method (ChangesHandler). No shared helper was created.
**Impact:** Low — code quality issue, not a bug. But if the event name format changes, 4 locations need updating.
**Mitigation:** Extract to a shared helper method. Deferred to Story 5.0.

### 4. REST Router Can't Handle Slashes in Document IDs
**What happened:** Story 4.3's HTTP test for `_design` filter initially failed because `/:db/:docid` route can't match `_design/test` (the slash splits it into two segments).
**Root cause:** `%CSP.REST` URL matching treats `/` as a segment separator. Design documents with IDs like `_design/myview` need dedicated routes (`/:db/_design/:designId`).
**Impact:** Workaround was simple (create design docs via DocumentEngine directly in tests). The actual `_design/` route support is a future epic (12+).
**Mitigation:** No action needed now. Epic 12 will add `_design/` routes.

## Preparation for Epic 5: Binary Attachment Management

### Dependencies on Epic 4

- `DocumentEngine.Save()` — will need attachment parameter added to the transaction
- `Storage.Changes` — attachment writes will generate changes feed entries (already handled)
- `$System.Event` signaling — attachment writes will trigger longpoll via existing DocumentEngine event

All prerequisites complete and stable.

### New Capabilities Needed

- `IRISCouch.Storage.Attachment` — new storage class for `^IRISCouch.Atts` via `%Stream.GlobalBinary`
- `IRISCouch.API.AttachmentHandler` — new handler for `GET/PUT/DELETE /{db}/{docid}/{attname}`
- MD5 digest computation for replication integrity
- `multipart/related` request parsing for inline attachment uploads
- Attachment stub handling (`"stub":true` for unchanged attachments during doc updates)
- Streaming I/O — must not buffer entire attachment in memory (NFR-P8)

### Risks

- **`%Stream.GlobalBinary` integration**: Need to understand IRIS stream API for binary storage and retrieval within transactions
- **Multipart/related parsing**: `%CSP.Request` may or may not support multipart parsing natively — needs research
- **Router route matching for 3-segment paths**: `/:db/:docid/:attname` — verify %CSP.REST supports this depth
- **Memory-bounded streaming**: Must verify that IRIS stream reads/writes don't buffer the full content
