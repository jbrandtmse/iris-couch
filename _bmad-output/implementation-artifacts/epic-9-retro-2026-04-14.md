## What Didn't Go Well / Challenges

### 1. $System.Security.Audit Requires Pre-Registered Security.Events
**What happened:** `$System.Security.Audit()` silently returns 0 (failure) unless the Source/Type/Name triple is pre-registered via `Security.Events.Create()` in %SYS namespace.
**Root cause:** IRIS documentation doesn't emphasize the registration requirement. The API fails silently — no exception, no error.
**Fix:** Added `EnsureEvents()` to register all 8 event types. Called from Installer.Install() on both fresh install and upgrade paths.
**Impact:** Without the fix, production installations would have zero audit trail with no visible errors.
**Lesson:** Always verify IRIS system APIs actually produce the expected effect, not just that they don't throw errors.

### 2. Audit Username Shows "UnknownUser" for HTTP Requests
**What happened:** The `username` field in %SYS.Audit events shows "UnknownUser" for all CSP/HTTP-originated requests.
**Root cause:** CSP gateway processes don't carry `$Username` through to the audit system. The authenticated user is captured in EventData JSON but not in the standard username field.
**Impact:** Operators querying audit log by username won't find IRISCouch events via the standard field. Must query EventData instead.
**Status:** Documented limitation — CSP gateway behavior, not fixable at application level.

### 3. Histogram Double-Counting Bug (Story 9.1)
**What happened:** The Prometheus endpoint renderer re-accumulated already-cumulative histogram bucket values, producing inflated metrics.
**Root cause:** The Collector stores cumulative values (each request increments ALL buckets where duration <= boundary), but the renderer assumed non-cumulative and added a running total.
**Fix:** Code review caught it and removed the re-accumulation.
**Lesson:** Prometheus histogram semantics require cumulative buckets — document this invariant in the Collector class.
