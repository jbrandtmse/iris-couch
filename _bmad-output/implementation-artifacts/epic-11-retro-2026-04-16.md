## What Went Well

1. **Compounding design-system reuse.** `TextAreaJson` (new in 11.3) was immediately reusable in 11.4 planning discussions and is a natural fit for Epic 12's view-source editor. `ConfirmDialog` gained a `warning` variant in 11.3 and was reused wholesale by 11.4's discard-on-nav-away flow. `JsonDisplay`, `PageHeader`, `Breadcrumb`, `FeatureError` all composed predictably with no modifications across all five stories.

2. **Task 0 pattern caught real backend defects before UI work began.** Story 11.3 Task 0 fixed a UrlMap ordering bug where `PUT /{db}/_design/{name}` was dispatching to the attachment handler. Story 11.4 Task 0 discovered `?deleted_conflicts=true` was never actually implemented despite the story spec assuming it existed. Both stories closed the loop with new ObjectScript HTTP tests. Prior epics would have shipped UI against a broken backend and debugged it later.

3. **Subscription-leak pattern never regressed.** Stories 11.3 and 11.4 each introduced components with TWO parallel request slots (save + delete, tree + body). Every slot had its own `activeRequest` variable, every save/delete swap cancelled the prior subscription, `ngOnDestroy` cleaned up both. Zero findings from code review on this axis — a direct win from Epic 10's codified rule.

4. **Full-Router integration test pattern for custom matchers (Story 11.4 review).** The initial `revisionsMatcher` ordering test merely compared array indices. Code review replaced it with a real `provideRouter` integration block that calls `Router.navigateByUrl('/db/foo/doc/bar/revisions')` and asserts the activated component — plus a negative-control test that reverses matcher order to prove ordering is load-bearing. This is the right pattern for every future custom matcher.

5. **Chrome DevTools MCP surfaced a real bug invisible to specs.** Story 11.4 dev noticed during browser verification that `?rev=<deleted-leaf>` returns 404. Tests were happy because the service's happy path never exercised a deleted leaf. Fix: `RevisionsService` now tolerates 404 on deleted-leaf branch fetches with a synthetic minimal response. Regression spec added. This is exactly the class of bug Epic 10's retrospective predicted manual verification would catch.

6. **Backend + UI co-evolution in single commit.** Both 11.3 (`DocumentHandler` UrlMap + `Router` wrappers) and 11.4 (`Storage.RevTree.GetDeletedConflicts` + handler clause) shipped backend and UI changes atomically with matching regression tests. No cross-repo drift, no feature-flag-gated hacks.

## What Didn't Go Well / Challenges

### Story-spec assumptions about backend capabilities

Story 11.4's spec said "backend already supports `?deleted_conflicts=true`" based on a surface read of `DocumentHandler.cls` lines 640–680. The actual implementation only supported `?revs_info=true` and `?conflicts=true`; the third was never wired up. Dev caught it during Task 0 but it could have caused mid-story churn.

**Preventive measure for Epic 12:** story creation should include a `curl` probe in the Tasks section (not just a read of the class) for any backend endpoint the UI depends on. Task 0 should actually execute that curl and paste the output into the story, not just describe the expected behavior.

### SideNav is now touched in every UI story

Stories 10.3 (created it), 11.1 (Design Docs entry), 11.2 (Security entry), 11.4 (Revisions entry with disabled-state). Four hardcoded entries in a single template, each story bolting on another with slightly different conditions. The disabled-when-no-docId pattern in 11.4 is clean but will become unwieldy when Epic 14 adds streaming-feed views.

**Recommended refactor:** Epic 12.0 or a standalone cleanup should extract the SideNav entries into a config array with typed `{label, route, icon, enabled: (ctx) => boolean}` records. Not a blocker; do when touching SideNav for the fifth time.

### Code-review agent startup failed 3 times (API 529 overloaded)

Cr-11-4 spawned 3 times with overloaded errors before the fourth attempt succeeded. Losses: roughly 60 seconds of retry churn and no observable data loss. Not a project-level issue; just ambient API capacity.

**Process note:** the /epic-dev-cycle skill could learn a short backoff between spawn retries. Currently the lead retries immediately on 529; a built-in 30s delay + three attempts with exponential backoff would be cheap insurance.

### Epic-10 deferred item still deferred

Network error messages duplicated across 3 components (`database-list.component.ts`, `database-detail.component.ts`, `document-detail.component.ts`) was flagged in Epic 10's retro as "LOW — DRY cleanup". It is still flagged in deferred-work. Story 11.4 added a fourth user (`revisions-view.component.ts`) which continues the copy-paste pattern rather than extracting a `classifyHttpError(err)` utility.

**This is a precisely the "what you agreed to fix last retro and didn't" accountability moment Amelia warns about.** Add to Epic 12.0 as non-deferrable.

## Key Insights

1. **Task 0 has earned its place as a gate for all beta/gamma UI stories.** When the UI is consuming a nontrivial backend surface, verify the surface FIRST with curl, not with a read-the-class assumption. Story 11.3 saved a day of UI debugging; Story 11.4 surfaced a backend gap that would have caused a mid-story pivot. The pattern works. Codify it.

2. **The subscription-leak rule paid for itself twice this epic.** Two components with two request slots each = four potential leak sites. Zero leaks in code review. The rule doesn't feel heavy; it feels like muscle memory now. That's the outcome a good process rule should produce.

3. **The `RevisionTree` component is now a template for "visually ambitious but bounded" UI work.** SVG + pure layout helper (`computeRevisionTreeLayout`) + CDK overlay popover + no external graph library + accessible keyboard navigation. The pure helper gets the heavy unit-test coverage; the component just renders. This split is reusable for any future data-visualization need (Epic 14 streaming feed timeline, perhaps).

4. **Chrome DevTools MCP is now the canonical second-pass check for UI stories.** Every Epic 11 story used it. One (11.4) caught a real backend bug. The 4–5 screenshots per story format is establishing itself as a de-facto AC — it's probably worth making this an explicit requirement in any future UI story template.

5. **Integration tests > structural tests for routing.** Array-index comparison tests passed when the routing was subtly wrong. The real-Router `navigateByUrl` + component assertion test catches the actual behavior. Generalizable to any Angular code where declarative config (route tables, `providedIn`, DI tokens) determines runtime behavior — test the behavior, not the config.

## Next Epic Preview — Epic 12: Pluggable JavaScript Runtime

### Scope

5 stories (12.1–12.5) introducing a `IRISCouch.JSRuntime.Sandbox` abstract interface with three backends: `None` (default, 501 responses), `Subprocess` (couchjs line protocol over `$ZF(-1)`), `Python` (embedded Python + QuickJS). Drives map/reduce views, `validate_doc_update`, and custom changes filters.

### Technology Switch

Epic 11 was Angular/TypeScript; Epic 12 is predominantly ObjectScript. The backend rules (`.claude/rules/iris-objectscript-basics.md`, `.claude/rules/object-script-testing.md`, `.claude/rules/research-first.md`) apply. Angular is irrelevant except where the admin UI surfaces JSRuntime status (which, per the 501-return pattern, is minimal — a `FeatureError` showing the verbatim `reason` string). No significant UI work planned.

### Dependencies on Epic 11

1. **Design-doc write path stability** (Story 11.3 Task 0) — Epic 12 will write and retrieve design docs extensively. The UrlMap fix and composite-ID handling must hold. Regression tests are in place.
2. **Design-doc body shape** (Story 11.1) — JsonDisplay-compatible shapes persist in `^IRISCouch.Docs`. Epic 12's JSRuntime.None must respect this (store without executing).

Both dependencies are stable; no blocking gaps.

### Preparation Required

1. **Read embedded Python manual** — `documentation/IRIS_Embedded_Python_Complete_Manual.md` (per project rules). Story 12.4 depends on correct Python bridge usage (`%SYS.Python`, `import iris` round-trip, availability detection).
2. **Read couchjs line protocol** — `sources/couchdb/share/server/` in the CouchDB source tree. Stories 12.2 and 12.3 implement client side of this protocol.
3. **Subprocess lifecycle research** — `$ZF(-1)` semantics for long-running processes, stdin/stdout buffering, process cleanup on IRIS shutdown. Perplexity MCP + InterSystems docs.

All preparation is research/reading; no code gating Epic 12.1's start.
