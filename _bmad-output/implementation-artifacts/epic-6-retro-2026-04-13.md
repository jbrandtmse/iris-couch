## What Didn't Go Well / Challenges

### 1. IRIS SQL Case Insensitivity — %EXACT() Required
**What happened:** Manual curl testing revealed document IDs returned as uppercase (`ORDER-001` instead of `order-001`) when querying through the Mango index-backed path. The full-scan path was unaffected because it reads directly from %Persistent objects.
**Root cause:** IRIS SQL is case-insensitive by default for string comparisons in both SELECT and WHERE clauses. All 13 SQL queries across 5 files needed `%EXACT()` wrapping: in SELECT to preserve case in returned values, and in WHERE for case-sensitive matching.
**Impact:** Required a post-review hotfix (`0a3928c`) touching 6 files. Would have been a P0 production bug if shipped.
**Mitigation:** This is a critical IRIS-specific pattern. Save to project rules and memory.

### 2. JSON null Rendering as String "null"
**What happened:** The `_all_docs` special index and `_explain` response rendered `"ddoc": "null"` and `"covering": "null"` as JSON strings instead of proper JSON null values.
**Root cause:** `%DynamicObject.%Set("key", "null", "null")` was used — the third parameter is the JSON type hint, but passing `"null"` as the value (first parameter) produces a string. Correct pattern is `%Set("key", "", "null")` where the empty string value is rendered as JSON null.
**Impact:** Fixed alongside the %EXACT() hotfix. Would cause client-side JSON parsing issues.
**Mitigation:** Document the `%Set("key", "", "null")` pattern for JSON null in project rules.

### 3. Selector Operator Semantics for Missing Fields
**What happened:** Story 6.2 code review found that `$ne` and `$nin` returned `false` when the field was missing from a document. CouchDB semantics require `true` for `$ne` on a missing field (a missing field is "not equal" to any value).
**Root cause:** The initial implementation treated missing fields as always-false without special-casing negation operators.
**Impact:** Auto-resolved during code review. Would have produced incorrect query results.
**Mitigation:** CouchDB selector semantics for missing fields are non-obvious. Document in project rules.
