## What Didn't Go Well / Challenges

### 1. DocumentHandler Grew to ~1000 Lines
**What happened:** DocumentHandler accumulated 9 handler methods (HandlePost, HandlePut, HandleGet, HandleDelete, HandleBulkDocs, HandleBulkGet, HandleAllDocs, HandleAllDocsPost, ValidateFields) reaching 1009 lines — double the 500-line target.
**Root cause:** All document-related endpoints were routed to the same handler class per the architecture's "per-subsystem handler" pattern, but the document subsystem is the largest in the system.
**Impact:** Code becomes harder to navigate and review. Adding Epic 4 changes feed endpoints would make it worse.
**Mitigation:** Split into BulkHandler and AllDocsHandler in Story 4.0.

### 2. Quit Inside While/For Loops in Try Blocks
**What happened:** Story 3.5's `new_edits=false` handler had `Quit` statements inside a `While` loop that only exited the loop, not the enclosing Try block. This caused a 400 error response to be followed by a 201 response (dual HTTP response).
**Root cause:** ObjectScript's `Quit` behavior is context-dependent — it exits the innermost block (loop, if, try) not necessarily the method.
**Impact:** Code review caught it, but it's a subtle bug that could recur.
**Mitigation:** Add coding guideline: use flag variables to control flow after errors in loops.

### 3. Storage Encapsulation Violations Keep Recurring
**What happened:** Stories 3.1, 3.5, and 3.6 all had direct `^IRISCouch.*` global access outside Storage.* classes. Code reviews caught all instances.
**Root cause:** Developers see nearby code (e.g., DocumentEngine) accessing globals and assume it's acceptable everywhere.
**Impact:** Minor — reviews catch it. But wastes review cycles.
**Mitigation:** Saved to memory. Story files must explicitly list which Storage.* methods to use. Created feedback memory `feedback_storage_encapsulation.md`.

### 4. Test Count Estimates Drifted
**What happened:** Story 3.0 estimated 57 existing tests but actual count was 48. This discrepancy cascaded through subsequent story estimates.
**Root cause:** Epic 2 retro reported 57 tests total, but some were added in the retro verification step after the last story count.
**Impact:** Negligible — test counts are informational, not blocking.
