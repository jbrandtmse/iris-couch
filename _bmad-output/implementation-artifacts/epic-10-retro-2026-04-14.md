## What Went Well

1. **Compounding component reuse** — tokens → primitives → features composed predictably. `PageHeader`, `Breadcrumb`, `ErrorDisplay`, `DataTable`, `ConfirmDialog`, `CopyButton` used across 3+ features with zero modifications.

2. **Accessibility-first discipline** — 26 axe-core assertions caught real regressions: warn color contrast (#B57B21 → #8B6914, 3.61 → 5.09), Badge 10px violation, missing aria-describedby. Keyboard-first (FocusKeyManager SideNav nav, FocusTrap dialogs, Escape/Enter/Space/`/`).

3. **Route state persistence via Angular router alone** — deep-linking, filter + pagination + scroll preserved via `onSameUrlNavigation`. No state-management library. Story 10.5 established the pattern; Story 10.6 inherited it.

4. **Prohibited-patterns audit clean** (Story 10.7) — no toasts, tours, dashboards, Material, relabeling, auto-refresh, masking, wizards, hover-nav, charts. The spec's negative space was respected.

5. **Verbatim error display pattern** (Story 10.3) established the principle; backend JSON `{error, reason}` surfaced unmodified. (Regressed in login; see Challenges.)

6. **Custom JSON tokenizer** (Story 10.6) — avoided Prism/Highlight.js. Keeps bundle at 125 KB and removes a supply-chain surface.

## What Didn't Go Well / Challenges

### Cross-story review patterns (recurring)

1. **Subscription/Observable leaks on rapid interaction** — Stories 10.3, 10.5, 10.6 each introduced the same bug (rapid clicks/pagination overlap, old sub not cancelled), caught only in code review. Formalize pattern: `activeRequest` tracking + unsubscribe-before-new + `ngOnDestroy` cleanup.

2. **Hardcoded colors in component CSS** — Stories 10.2, 10.3, 10.4. Hex and RGBA leaked past the design-tokens layer (destructive hover, Badge alpha backgrounds). Tokens exist; they were forgotten.

3. **Network error UI duplicated across 3 components** (Story 10.7). DRY cleanup deferred; now partially resolved (see `mapError` below) but 3 sites still contain inline copies.

### Manual verification gap — the critical systemic lesson

Story 10.7's "cross-browser verification" AC passed in checklist form but **never clicked "Sign in"**. End-to-end browser exercise via Chrome DevTools MCP during this retrospective uncovered **five production-affecting bugs invisible to unit tests, axe-core, and the TESTING-CHECKLIST**:

1. **Dev proxy entirely dead.** `proxy.conf.json` was untracked (`??` in git) and `angular.json` had no `proxyConfig` key — `ng serve` never loaded it. Even if loaded, the JSON context keys (`/_session`) didn't match the base-href prefix `/iris-couch/_utils/`.

2. **Login error rephrased, not verbatim.** Bad creds always showed `"unauthorized — Name or password is incorrect."` regardless of real status. A 404 (proxy failure) displayed as "404 unauthorized…" — directly violates Story 10.3's verbatim-error principle.

3. **Database list crashed whenever non-empty.** `DatabaseService.listAllWithInfo` at line 73 read `info.sizes.file`. IRISCouch returns the legacy flat `disk_size`, no `sizes` object. TypeError in RxJS `map` surfaced as generic `"unknown / An unexpected error occurred"`. **This is the feature's golden path.**

4. **Generic error fallback masked JS exceptions.** `err.error && typeof err.error === 'object'` pattern at 5 sites caught thrown non-HTTP `Error` instances (like Bug 3's TypeError) and relabeled them as server errors, hiding the real bug from developers.

5. **Favicon 404** — `/favicon.ico` 404 on every page (relative-path issue with base-href).

All five fixed during this retrospective; see "Post-Retro Fixes" below.

### Backend/frontend contract gaps (discovered via browser testing)

- `GET /{db}` response missing `sizes: {file, external, active}` object (CouchDB 2.x+ spec). IRIS returns only legacy `disk_size`.
- `GET /{db}` returns `update_seq` as number; CouchDB spec is string sequence ID.
- `GET /{db}` returns `disk_size: 0` even for databases with content — backend size reporting unimplemented.

### Deferred from Epic 10 stories

- **Per-row delete UI** on database list (Story 10.4 AC #5) — DataTable designed for table-level actions; needs action-column extension.
- **Design doc ID encoding** (Story 10.6) — `_design/myapp` needs special URL handling; unblocks Story 11.1.

## Key Insights

1. **Unit tests + axe-core ≠ cross-browser verification.** Story 10.7 claimed cross-browser verification but never exercised login through the dev server. A 60-second click-through would have failed on Bug 1. **Axe-core verifies component a11y in isolation; it does not verify that a component successfully loads.**

2. **Tolerant services survive backend evolution.** The frontend should tolerate both `sizes.file` and legacy `disk_size` — CouchDB itself does. Strict shape assumptions introduce coupling that breaks silently when the backend lags.

3. **Generic error fallbacks hide real bugs.** `{error: 'unknown', reason: 'An unexpected error occurred'}` is worse than no handling — it catches TypeErrors and presents them as server faults. Distinguish `HttpErrorResponse` from `Error`.

4. **Proxy config should be exercised in CI.** The untracked `proxy.conf.json` would have been caught by any CI job that ran `ng serve` + a smoke test.

5. **Dev environment parity matters.** IRIS serves the built UI under `/iris-couch/_utils/` in production; dev server serves it the same way but proxies differently. Any base-href vs proxy mismatch breaks dev without failing prod — precisely the configuration that escapes local testing.
